### 6.2 v2 → v3 Fix

#### ESCALATE Logic Error

**Problem (v2):**
```python
if confidence < 0.3:  # Low-signal
    rule = HOLD  # Catches everything <0.3
elif confidence < 0.2:  # Critical uncertainty
    rule = ESCALATE  # NEVER REACHED
```

**Root Cause:** Order of if/elif checks

**Fix (v3):**
```python
if confidence < 0.2:  # Critical uncertainty - CHECK FIRST
    rule = ESCALATE
elif confidence < 0.3:  # Low-signal
    rule = HOLD
```

**Impact:**
- ESCALATE rate: 0% → 18.2%
- Revenue conflict (15% confidence) → Correctly escalated
- Legal liability (18% confidence) → Correctly escalated
- Overall rating: 4.7/5 → 5.0/5 (+0.3)

**Lesson:** Order of conditional checks matters in protocol enforcement.

### 6.3 Cross-Version Lessons

#### Lesson 1: Source Quality > Source Quantity (to a point)

**Finding:**

**Recommendation:** ALWAYS include contrarian view with historical precedents.

#### Lesson 3: ESCALATE Protocol Protects Against Legal/Ethical Risk

**Finding:**

- v2: ESCALATE broken → Legal claims misrouted to HOLD
- v3: ESCALATE working → Legal claims flagged for counsel review

**Recommendation:** ESCALATE protocol is critical for compliance (EU AI Act Article 10, SEC disclosure rules).
